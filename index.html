❓ FAQ — Customer asks about a missing feature or changed behavior

Why can't I remove a specific collaborator like I could in MDA? ▶

What changed: MDA's "Remove specific collaborator" governance action has no direct equivalent in Purview DLP.

What to do: Use Purview DLP to restrict access (block external users broadly), then supplement with a Power Automate flow triggered by the DLP alert to remove specific sharing permissions via SharePoint/OneDrive APIs.

→ See: Parity Depth Library · Governance Actions mapping

Where is user quarantine in Purview? MDA had user quarantine + admin quarantine. ▶

What changed: Purview DLP supports admin quarantine only. User quarantine (move file to user's quarantine folder) has no direct equivalent.

What to do: Use Purview DLP to restrict access (block-in-place), then add a Power Automate flow to move the file to a designated quarantine folder.

→ See: Parity Depth Library · Quarantine row

My MDA policy applied a sensitivity label. Why do I need two Purview policies now? ▶

What changed: In MDA, labeling was a governance action inside the file policy. In Purview, label application is handled by a separate auto-labeling policy (Information Protection), not by DLP.

What to do: Create two policies:

  • A Purview DLP policy for detection and restriction/alerting
  • A Purview auto-labeling policy (purview.microsoft.com → Information Protection → Auto-labeling) for label application

→ Key Resources: Auto-labeling Policies ↗

MDA could trash / delete a file. Where is that in Purview? ▶

What changed: Purview DLP has no "trash file" or "delete file" action.

What to do: Use a Purview DLP policy to restrict access, then trigger a Power Automate flow on the DLP alert to delete the file programmatically.

→ See: Parity Depth Library · Gap actions

MDA scoped policies to specific folders. Why can't I do that in Purview? ▶

What changed: MDA supported "Parent folder" as a file filter. Purview DLP scopes at the site level (SharePoint site URLs), not the folder level.

What to do: Scope the Purview DLP policy to the relevant SharePoint site. If folder-level control is critical, evaluate SharePoint-level permissions or Power Automate as a supplement.

My MDA policy monitored files in AWS / ServiceNow / Webex. Does Purview cover those? ▶

What changed: Purview DLP does not support AWS, ServiceNow, or Cisco Webex. There is no equivalent location in Purview DLP for these apps.

What to do: Evaluate native DLP capabilities in those platforms. These are confirmed gaps — do not promise a roadmap date without PG confirmation.

Supported third-party apps in Purview: Box, Google Workspace, Dropbox, Salesforce (via "Instances" location).

→ See: DLP for Non-Microsoft Cloud Apps ↗

MDA had no simulation mode. How does the Purview simulation work? ▶

What's new (net gain): Purview DLP has full simulation mode ("Test it out first") — policies run and log matches without taking any enforcement action. Run for 1–2 weeks before going live.

What to do: Always start new Purview policies in simulation mode. Review matches in the DLP Alerts dashboard and Activity Explorer, then compare against MDA match volumes before enforcing.

→ Navigation: purview.microsoft.com → Data loss prevention → Policies → [Policy] → Simulation results

MDA had a 50 file policy limit. What are the Purview limits? ▶

What's new (net gain): Purview has significantly higher limits — 10,000 MIP + MIG policies and 600 DLP rules per tenant (vs. MDA's 50 file policies).

⚡ Escalation & Routing — Where does this go?

🔴 Something is broken
GA behavior that worked before is no longer working — DLP not enforcing, labels applying incorrectly, data missing from Content Explorer.
→ Route to Purview CSS (hand off case)
MDA CSS confirms it's a Purview-side break, then transfers. You do not open the ticket — you hand off.
🟡 Feature gap / customer wants more
Customer asks for a capability Purview doesn't have yet — remove specific collaborator, folder scoping, AWS app support, user quarantine.
→ Product feedback (NOT a support ticket)
Log via Purview DLP Product Team field feedback intake. Acknowledge the gap clearly to the customer — do not promise a date.
